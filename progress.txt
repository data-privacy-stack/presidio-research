## Codebase Patterns
- Branch: ralph/dataset-anchored-mapper (created from ralph/presidio-evaluator-redesign)
- Tests run with: pytest tests/entity_mapping/ (fast); full suite: pytest
- Linting: ruff check presidio_evaluator/entity_mapping/
- EntityHierarchy._strip_bio() is a static method used both in hierarchy.py and mapper.py
- canonical_to_branch[entity] gives the full branch path list; len() gives depth
- all_canonical_entities is the set of nodes at canonical_depth
- _rebuild() recomputes raw_to_canonical, all_canonical_entities, canonical_to_branch
- _build_branch_map uses canonical_depth but also records intermediate nodes with their path
- The full hierarchy tree is in presidio_evaluator/entity_mapping/definitions.py (HIERARCHY dict)
- DO NOT add overlap_counts to MappingIssue as a positional field - use field(default_factory=...)
- pre-commit runs: ruff format (THEN ruff check, THEN pytest unit tests). Always run ruff format before committing.
- LOCATION is depth-2 (PII->LOCATION), so it becomes COLLISION_AMBIGUOUS at eval_depth=3. Use LOC (depth-3) instead.
- US_PASSPORT resolves via EXACT (it's in the alias map), not COUNTRY tier. Use GERMANY_PASSPORT_NUMBER to test COUNTRY tier.
- Presidio outputs depth-2 labels (PERSON, LOCATION, DATE_TIME, ORGANIZATION). These need .map() to depth-3 before get_mapped_results_dataframe() at eval_depth=3.
- get_mapped_results_dataframe() returns 7 columns (adds original_annotation, original_prediction). Shape assertions must compare row count only.
- eval_depth computed from annotation labels ONLY (self._label_annotation_counts), not predictions.
